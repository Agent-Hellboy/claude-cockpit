Architecture · Go · Claude Code

claude-cockpit / design

A single Go binary that hangs two instruments off Claude Code: a live status line, and an advisor that suggests the next useful control before a session gets expensive, repetitive, or hard to steer. It never automates your session.

01The cockpit metaphor

The design is organized like an aircraft cockpit, and the advisor's prompt says so verbatim. Gauges show context fill, model, rate/cost pressure, and search patterns. Warnings fire when context runs hot or an expensive model is overused. Controls are the levers it suggests — /compact, /model, skills, subagents, MCP, graphify. And a checklist ethos governs the whole thing: prefer reversible, local actions, and ask before anything risky. The instruments report; the pilot still flies.

02How a turn flows through it

Two Claude Code hooks drive everything. Both return fast; the expensive work runs detached.

Every render · statusLine hook

Claude Code → JSON payload → cockpit statusline → renders the two-row bar + writes .cockpit-state

The state write is the key bridge: the status line is the only place Claude Code hands over the real context window, cost, and rate-limit numbers. It snapshots them to disk so the analyzer — which isn't given that data — can read the truth instead of guessing.

Every turn end · Stop hook

Stop event → cockpit analyze → cadence gate (skip most turns) → collect signals → spawn detached worker

analyze returns immediately so the turn never waits. It bumps a per-session turn counter and only proceeds on a cadence — every 10th turn early, every 5th after turn 10, every 2nd after turn 25 — then hands a plain-text signals file to a fully detached process (own process group, /dev/null fds).

Detached · the worker

Phase 1 · claude -p --model haiku → emoji suggestion lines → if TOOLGAP flagged → Phase 2 · WebSearch for a real tool → write .session-report + .model-hint

The report is read back by the next status-line render and shown as wrapped yellow rows — closing the loop. Two phases keep it cheap: a local, tool-less advisor on every run, and a web search only when phase 1 detects an external capability with no matching MCP server.

03The two-phase advisor

A cheap model reads instruments and recommends controls by name. Output is filtered structurally, not by trusting the model to behave.

Phase 1 — local advisor

worker.go · instr

haiku with no tools reads the SIGNALS block and returns 1–3 emoji-led sentences: model down-shifts, /compact when context ≥ 75%, budget protection, skills/subagents/MCP by name, graphify over grep.

Phase 2 — tool discovery

worker.go · searchInstr

Runs only if phase 1 emits a TOOLGAP: line. A focused WebSearch finds one current, popular Claude Code integration for the gap — phrased as an "audit first" suggestion, never auto-installed.

Structural filtering

util.go · emojiLines

Keeps only lines that begin with a non-ASCII rune and don't end in a colon — so prose, preambles, and headers are dropped regardless of which emoji the model picked. Markdown bold is stripped.

05What the analyzer measures

Cheap signals parsed from the tail of the transcript (last 3,000 lines) plus the live state snapshot — no extra model calls to gather them.

Pressure

Real context window + fill %, session cost, 5h / 7d rate-limit usage — read from the status line's .cockpit-state, falling back to inferring the window from the model name.

Behavior

Tool histogram, manual search count (Grep / rg / find), and files read 3+ times — the redundancy that signals a better approach exists.

Capability

Installed skills, subagents, MCP servers, and plugins discovered on disk; graphify graph presence; and a source-file count with a build ETA when it's absent.

07Design principles

Advisory

It never steers for you. Cockpit shows the control and the reasoning; you choose whether to pull the lever. Nothing is auto-installed or auto-switched.

Non-blocking

The turn never waits. Hooks return in milliseconds; the model call runs in a detached process so the advisor can't slow your session.

Cheap

Spend nothing you don't have to. Cadence throttling, a tool-less phase 1, and web search only on a real gap keep the running cost near zero.

Self-contained

One small binary. No Go, jq, or runtime dependency at install; settings are merged in Go and backed up, foreign hooks preserved.

Private

Your keys, your model. Signals go only through your own claude -p invocation; secrets in prompts are redacted; prompt text can be omitted entirely.

Testable

Rendering is pure. renderStatusline does no IO, so the bar — gauges, colors, wrapping, thresholds — is unit-tested without a live session.
